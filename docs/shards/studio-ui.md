# Studio UI Shard

## Purpose

Review, edit, validate, test, and export workflows visually later.

## Scope

Canvas, inspector, validation panel, test chat, export panel.

## Current Decisions

- Studio comes after runtime/API proof.
- Xyflow/React Flow is a likely candidate later.
- UI should be calm and operational, not decorative.

## Do Not Do

- Do not build UI before core semantics are proven.
- Do not create a landing page as the primary experience.
- Do not make UI the source of workflow truth.

## Relevant Future Tasks

TASK-009.

## Acceptance Principles

The UI must make workflow behavior easier to inspect and fix.
