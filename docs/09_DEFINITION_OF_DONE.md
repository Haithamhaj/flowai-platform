# 09 Definition Of Done

A task is done only if:

- Acceptance criteria are met.
- Tests/build were run or explicitly documented as not run.
- Changed files are listed.
- Skill/MCP check result is documented.
- Risks are documented.
- Docs are updated if needed.
- Handoff is written.
- No unrelated changes are included.
- Any decision is recorded in `docs/10_DECISION_LOG.md`.
- Security-sensitive changes were reviewed explicitly.
